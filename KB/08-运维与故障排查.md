# 运维与故障排查

[← 返回目录](./index.md)

> 企业场景下的日常运维、日志与备份、常见问题与应急处理。

---

## 日常运维

- **日志**：`openclaw logs --follow` 实时查看；建议将网关日志接入现有日志系统，便于审计与排查。
- **网关部署**：建议部署在内网服务器，以 `openclaw gateway install` 常驻运行；仅允许受信任的飞书事件来源（长连接由飞书官方 SDK 建立）。
- **更新与备份**：定期更新 OpenClaw 版本；备份 `~/.openclaw/openclaw.json` 时注意脱敏或加密存储，避免 App Secret 泄露。

## 常见问题与排查

### 群里不回复

1. 看日志：`openclaw logs --follow`，在群里 @ 机器人观察是否有入站。
2. 确认 `groupPolicy` 不是 `"disabled"`，且该群在 `groupAllowFrom` 中（若使用 allowlist）。
3. 默认需 **@ 机器人** 才回复；确认机器人已加入该群。

### 收不到消息（网关侧无入站）

1. 看日志并发一条飞书消息，确认是否有记录。
2. 确认网关在跑：`openclaw gateway status`。
3. 检查飞书应用**权限**、**长连接**（WebSocket）、事件订阅 `im.message.receive_v1`、应用已**发布并审核通过**（见 [飞书应用创建与发布](./03-飞书应用创建与发布.md)）。

### App Secret 泄露

1. 立即重启网关：`openclaw gateway restart`。
2. 在配置中更新 App Secret（`openclaw.json` 或环境变量）。
3. 在飞书开放平台该应用下**重置 App Secret**。  
   详见 [安全与合规](./06-安全与合规.md)。

### 发送失败（机器人发不出消息）

1. 看日志中的具体报错。
2. 确认应用已发布且具备 `im:message:send_as_bot` 权限。

## 通用诊断命令

| 命令                        | 用途           |
| ------------------------- | ------------ |
| `openclaw gateway status` | 网关是否在运行      |
| `openclaw logs --follow`  | 实时日志，排查收发与错误 |
| `openclaw health`         | 健康检查         |
| `openclaw doctor`         | 环境与配置诊断      |

更多问题可查阅 [OpenClaw 官方故障排查](https://docs.openclaw.ai/help/troubleshooting) 与 [渠道故障排查](https://docs.openclaw.ai/channels/troubleshooting)。  
访问控制与允许列表见 [通道配置与访问控制](./04-通道配置与访问控制.md)；凭证与审计见 [安全与合规](./06-安全与合规.md)。
