# 安装与运行

[← 返回目录](./index.md)

> 企业内部署 OpenClaw 网关的环境要求、安装方式与网关管理。建议将网关部署在内网或可控云环境，便于访问控制与审计。

---

## 环境要求

- **Node.js**：22 或更高（推荐 24）
- **系统**：macOS、Linux，或 Windows（建议 WSL2）
- **资源**：约 2GB 内存、500MB 磁盘；企业长期运行建议预留更多资源并做监控

---

## 安装 OpenClaw

### npm（推荐）

```bash
npm install -g openclaw@latest
```

### pnpm

```bash
pnpm add -g openclaw@latest
```

---

## 首次配置与启动

### 新手向导（推荐）

```bash
openclaw onboard --install-daemon
```

可一次性配置网关、模型、工作区及渠道。向导中可选择**飞书**并填写 App ID、App Secret（需先在飞书开放平台创建企业自建应用，见 [飞书应用创建与发布](./03-飞书应用创建与发布.md)）。

### 仅添加飞书渠道

若已完成初次安装，只需添加飞书：

```bash
openclaw channels add
```

选择 Feishu，输入 App ID 和 App Secret。

## 网关管理常用命令


| 命令                                       | 说明                               |
| ---------------------------------------- | -------------------------------- |
| `openclaw gateway status`                | 查看网关状态                           |
| `openclaw gateway`                       | 前台启动（默认端口 18789）                 |
| `openclaw gateway install`               | 安装并启动守护进程（LaunchAgent / systemd） |
| `openclaw gateway restart`               | 重启网关服务                           |
| `openclaw gateway stop`                  | 停止网关服务                           |
| `openclaw logs --follow`                 | 实时查看网关日志                         |
| `openclaw pairing list feishu`           | 列出飞书待批准配对                        |
| `openclaw pairing approve feishu <CODE>` | 批准指定配对码                          |


企业环境建议：将网关部署在内网服务器，通过 `openclaw gateway install` 以服务方式常驻，并将日志接入现有日志系统便于审计（见 [运维与故障排查](./08-运维与故障排查.md)）。

## 使用与命令

### 正常聊天

1. 确保网关运行：`openclaw gateway status`。
2. 在飞书中找到机器人：私聊或拉入群聊；企业若使用 allowlist，需已在该列表中（见 [通道配置与访问控制](./04-通道配置与访问控制.md)）。
3. 私聊若为配对模式：根据机器人回复的配对码执行 `openclaw pairing approve feishu <CODE>`。
4. 群聊中需 **@ 机器人**（默认）才会触发回复。

### 飞书内斜杠命令（文本）

飞书暂不支持原生命令菜单，以下以**纯文本**发送：


| 命令        | 说明      |
| --------- | ------- |
| `/status` | 查看机器人状态 |
| `/reset`  | 重置当前会话  |
| `/model`  | 查看或切换模型 |

## 访问控制面板

OpenClaw 服务器默认关闭远程访问，只能本地访问，因此需要通过 SSH 隧穿访问。

1. 打开本地电脑，例如Windows打开控制台cmd，执行隧穿命令

```bash
ssh -L 18789:127.0.0.1:18789 root@你的服务器IP
```

其中root、服务器IP要替换成你自己的服务器和管理员账号，系统不同管理员账号可能不同，比如ubuntu服务器的管理员账号就是ubuntu

2. 打开服务器端命令行，获取访问令牌及地址

```bash
openclaw dashboard --no-open
```

该命令将返回访问服务器的令牌，拷贝该返回地址
