## 企业应用是什么

在本手册中，**企业应用**指：在企业内使用 OpenClaw，通过**飞书**等渠道为员工提供统一的 AI 助手入口，并做好**访问控制、多团队隔离、安全与合规、运维审计**的完整方案。

- **OpenClaw**：自托管 AI 网关，数据与推理在你控制的机器上。
- **飞书**：作为主要接入渠道，使用企业自建应用，由企业管理员创建、审批与发布。
- **企业视角**：谁能用、用在哪些群、不同部门是否用不同 AI 能力、如何审计与应急，均需在规划时考虑。

## 落地时需考虑的 10 个角度

| 角度             | 要点                                                        | 本手册章节                                             |
| -------------- | --------------------------------------------------------- | ------------------------------------------------- |
| **访问与权限**      | 谁可以私聊/群聊、哪些群可 @ 机器人、是否按部门/角色限制；配对 vs 允许列表（企业推荐 allowlist） | [通道配置与访问控制](./04-通道配置与访问控制.md)                    |
| **安全与凭证**      | App Secret / API Key 保管、配置文件权限、泄露应急、飞书应用最小权限              | [安全与合规](./06-安全与合规.md)                            |
| **多团队 / 多能力**  | 不同部门或群使用不同 Agent（工作区/模型），按人/群路由到客服、技术等场景                  | [多团队与多 Agent](./05-多团队与多Agent.md)                 |
| **技能（Skills）** | 安装哪些技能、优先国内可用的企业级技能、API 与密钥管理、高危技能审批                      | [Skills 与企业能力](./07-Skills与企业能力.md)               |
| **合规与审计**      | 谁在何时使用了机器人、日志留存与检索、满足内部或行业合规要求                            | [安全与合规](./06-安全与合规.md)、[运维与故障排查](./08-运维与故障排查.md) |
| **部署与高可用**     | 网关部署在内网或云、单点或多实例、重启与更新策略                                  | [安装与运行](./02-安装与运行.md)、[运维与故障排查](./08-运维与故障排查.md) |
| **数据与隐私**      | 对话与会话数据存放位置、是否经境外 AI 服务、飞书侧数据留存策略                         | [安全与合规 - 自托管与数据](./06-安全与合规.md)                   |
| **成本与用量**      | AI 模型调用成本、飞书 API 配额、技能侧第三方 API 用量与限流                      | [配置参考](./10-配置参考.md)及模型/渠道文档                      |
| **渠道生命周期**     | 飞书企业自建应用的创建、权限、发布、停用与回收流程                                 | [飞书应用创建与发布](./03-飞书应用创建与发布.md)                    |
| **运维与应急**      | 日志、监控、备份、故障排查与回滚、Secret 泄露等应急预案                           | [运维与故障排查](./08-运维与故障排查.md)、[安全与合规](./06-安全与合规.md) |

**建议落地顺序**：先定**访问范围与安全基线**（谁用、允许列表、凭证管理），再定**部署方式与日志审计**，最后按部门需求配置**多 Agent 与 Skills**。

## 本手册结构说明

- **02 安装与运行**：环境与网关部署、日常使用与命令速查，为后续所有能力的基础。
- **03 飞书应用创建与发布**：飞书侧企业自建应用的完整生命周期。
- **04 通道配置与访问控制**：在 OpenClaw 中接飞书并做企业级访问控制（allowlist、群策略）。
- **05 多团队与多 Agent**：按部门/群分配不同 Agent，实现能力隔离与场景化。
- **06 安全与合规**：凭证、审计、数据与合规要点。
- **07 Skills 与企业能力**：国内可用的企业级技能获取与安装。
- **08 运维与故障排查**：日常运维与应急。
- **10 配置参考**：飞书通道配置项速查。
