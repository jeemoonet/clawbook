# feishu-calendar：飞书日历自动化

[← 返回技能列表](./index.md)

---

## 一句话用途

对飞书日历做**创建/管理日程、查询空闲/忙碌、订阅日历变更**，基于飞书 Calendar v4 API，适合**会议排期、冲突检测、与会议室 Pad 联动**等自动化场景。

---

## 适用场景

- **会议排期**：根据参与人空闲时间推荐会议时间、自动创建日程。
- **冲突检测**：创建前检查是否与已有会议冲突。
- **日历同步与订阅**：订阅日历变更事件，与其他系统联动（如会议室展示屏、OA）。

与 [feishu-calendar-tool](./feishu-calendar-tool.md) 相比，本技能更偏「自动化与集成」，能力更完整（空闲/忙碌、订阅等）。

---

## 安装

```bash
clawhub install feishu-calendar
# 或（若 slug 为 feishu-calender，以搜索为准）
skillhub install feishu-calendar
```

---

## 配置

- **凭证**：与 OpenClaw 飞书通道共用 `channels.feishu` 中的 appId、appSecret；或按技能文档配置环境变量 `FEISHU_APP_ID`、`FEISHU_APP_SECRET`。
- **飞书开放平台**：应用需具备日历相关权限（日程的创建、读取、更新、删除及空闲忙碌查询、订阅等），具体权限名以 [飞书日历 API](https://open.feishu.cn/document/server-docs/calendar-v4/overview) 为准。

## 怎么用

### 在对话中使用

用户可在飞书中说；AI 会调用本技能的日历工具执行查询或创建，并回复结果。

### 三个示例

**示例 1：查多人空闲时间**

- **用户**：「帮我看下张三和李四下周二下午谁有空。」
- **Agent 实现**：解析出参与人（张三、李四）、时间（下周二下午）→ 调用 feishu-calendar 的**空闲/忙碌查询**接口，传入两人在飞书中的 user_id/open_id 及时间范围 → 技能向飞书 Calendar API 请求各人该时段忙闲 → 返回「张三 14:00–15:00 空闲；李四 14:00–16:00 均忙」等 → Agent 整理成「下周二下午：张三有空，李四该时段有会」并可选推荐「可约 15:00 后两人都空」。
- **最终结果**：用户得到两人在该时段的空闲情况，便于定会。

**示例 2：创建带参与人的会议**

- **用户**：「给咱们组约一个下周三 14:00–15:00 的周会，参与人 @张三 @李四。」
- **Agent 实现**：解析出时间、标题（周会）、参与人 → 调用 feishu-calendar 的创建日程接口，传入 start/end、title、attendees（张三、李四的 user_id）→ 技能请求飞书日历 API 创建日程并发送会议邀请 → Agent 回复「已创建周会，下周三 14:00–15:00，已邀请张三、李四」。
- **最终结果**：飞书日历中新增该日程，张三和李四收到会议邀请，用户收到确认。

**示例 3：查明日会议列表**

- **用户**：「我明天有哪些会？」
- **Agent 实现**：解析为「当前用户、明日、列表日程」→ 调用技能的列表接口，传入明日 0:00–23:59（或用户时区）→ 技能返回该用户明日所有日程 → Agent 整理成「明天你有 3 个会：9:00 站会，10:00 需求评审，15:00 客户沟通」并附时间与标题。
- **最终结果**：用户收到明日会议清单，无需打开日历查看。

### 典型能力

- **创建/更新/删除日程**：标题、时间、参与人、地点、描述等。
- **空闲/忙碌查询**：按用户与时间范围查询，用于推荐会议时间。
- **订阅**：订阅日历变更事件（需在飞书侧配置并处理回调），便于与其他系统同步。

---

## 注意事项

- 参与人需为飞书用户（通常以 user_id 或 open_id 形式传入）；跨租户场景以飞书文档为准。
- 会议室资源若通过日历管理，也需相应权限与配置。
