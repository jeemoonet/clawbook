# feishu-calendar-tool：飞书日历（零配置）

[← 返回技能列表](./index.md)

---

## 一句话用途

用**当前 OpenClaw 里已配置的飞书凭证**，直接对飞书日历做**创建、列表、删除日程**，无需再配环境变量，适合「在飞书里跟机器人说一声就查日程、约会议」。

---

## 适用场景

- 在飞书里问：「我明天有什么会？」—— 列出明日日程。
- 「帮我把下周一下午 3 点约一个产品评审会，1 小时」—— 创建一条日程。
- 「把刚才那个会删掉」—— 删除指定日程。

适合个人或小团队在飞书中用自然语言管理日历，无需切到飞书日历 App。

---

## 安装

```bash
clawhub install feishu-calendar-tool
# 或
skillhub install feishu-calendar-tool
```

---

## 配置

**零配置**：技能会从 `~/.openclaw/openclaw.json` 的 `channels.feishu.accounts` 中读取 appId、appSecret，与飞书通道共用同一套应用。

若你的 OpenClaw 已能正常在飞书收发消息，一般**无需额外配置**。只需在飞书开放平台为该应用申请**日历相关权限**（如查看、创建、删除日程），并发布应用。

---

## 怎么用

### 在对话中使用

用户直接在飞书里对机器人说即可；AI 会调用本技能的日历工具完成对应操作并回复结果。

### 三个示例

**示例 1：查本周日程**

- **用户**：「查一下我本周的日程。」
- **Agent 实现**：解析为「按时间范围列日程」→ 调用 feishu-calendar-tool 的列表接口，传入本周起止时间（或技能默认「本周」）→ 技能用 openclaw.json 中的飞书凭证请求日历 API，必要时先解析日历 ID 并缓存 → 返回日程列表（标题、时间、地点等）→ Agent 整理成「本周你有 X 个日程：周一 10:00 …」回复。
- **最终结果**：用户收到本周所有日程的简短列表，无需打开飞书日历 App。

**示例 2：创建会议**

- **用户**：「明天下午 2 点创建一个会，标题叫需求评审，1 小时。」
- **Agent 实现**：解析出开始时间（明日 14:00）、时长（1 小时）、标题 → 调用 feishu-calendar-tool 的创建接口，传入标题、start/end（或 start + duration）→ 技能向飞书日历 API 发起创建请求 → 返回新日程 ID 或链接 → Agent 回复「已创建：需求评审，明天 14:00–15:00」。
- **最终结果**：用户飞书日历中新增一条「需求评审」，并收到确认消息。

**示例 3：删除指定日程**

- **用户**：「把我 3 月 15 号上午 10 点那个会删掉。」
- **Agent 实现**：解析出时间与意图「删除」→ 先调用列表接口查 3 月 15 日上午 10 点左右的日程，匹配到唯一一条（或让用户二选一）→ 调用删除接口传入该日程 ID → 技能请求飞书日历 API 删除 → Agent 回复「已删除该日程」。
- **最终结果**：对应日程从日历中移除，用户收到删除确认。

### 能力说明

- **创建日程**：支持标题、开始/结束时间、描述、地点、参与人、时区等（以技能实际参数为准）。
- **列表**：按时间范围查询日程；技能内部会做日历 ID 的发现与缓存（如 24 小时缓存），减少重复请求。

---

## 注意事项

- 日历权限需在飞书开放平台勾选并生效；若提示无权限，检查应用权限与发布状态。
- 多日历账号时，技能通常使用默认或主日历；若有多个飞书应用账号，以技能文档为准是否支持指定账号。
