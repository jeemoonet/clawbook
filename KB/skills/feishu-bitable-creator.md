# feishu-bitable-creator：飞书多维表格创建

[← 返回技能列表](./index.md)

---

## 一句话用途

**从零创建**飞书多维表格：按你定义的字段和初始数据，自动建表、清占位内容、批量写入记录，适合快速搭工单表、项目表、统计表等。

---

## 适用场景

- **快速建表**：例如「帮我建一个需求表，要有标题、负责人、状态、截止日期、备注」。
- **模板化表格**：固定字段结构，多次创建同类表格（如每个项目一个任务表）。
- **与 [feishu-api-bitable](./feishu-api-bitable.md) 配合**：creator 负责「建表」，api-bitable 负责日常「增删改查」。

---

## 安装

```bash
clawhub install feishu-bitable-creator
# 或
skillhub install feishu-bitable-creator
```

---

## 配置

### 1. 飞书凭证

与 OpenClaw 飞书通道共用 `channels.feishu` 的 appId、appSecret；或按技能要求配置环境变量。

### 2. 飞书开放平台权限

需具备：

- 多维表格应用权限（如 `bitable:app`）
- 云文档/驱动相关权限（如 `drive:permission:manage`、`drive:drive:read`），用于在指定知识库或空间创建表格

具体权限名以技能 SKILL.md 和 [飞书开放平台](https://open.feishu.cn/app) 为准。配置后需重新发布应用。

## 怎么用

### 在对话中使用

用户可在飞书中说；AI 会调用本技能在指定位置创建多维表格，按描述创建字段并可选写入初始记录；技能通常会**自动清理**飞书默认的占位行列，使表格可直接使用。

### 三个示例

**示例 1：新建项目任务表**

- **用户**：「帮我新建一个「项目任务」多维表格，字段：任务名、负责人、状态（未开始/进行中/已完成）、截止日期。」
- **Agent 实现**：解析出表名「项目任务」、字段列表及类型（任务名-文本、负责人-人员、状态-单选、截止日期-日期）→ 调用 feishu-bitable-creator，传入创建位置（如默认知识库或用户指定目录）及字段定义 → 技能在飞书中创建多维表格、创建表与列（并清理占位行列）、可选写入 0 行或 1 行示例 → 返回表格链接或 app_token+表 id → Agent 回复「已创建「项目任务」表，含任务名、负责人、状态、截止日期，链接：…」。
- **最终结果**：用户飞书中出现一张新的多维表格，列已就绪，可直接开始填任务。

**示例 2：在知识库里建周报表**

- **用户**：「在咱们组的知识库里建一个「周报」表，字段：姓名、本周完成、下周计划、备注。」
- **Agent 实现**：解析出创建位置「某知识库」、表名「周报」、字段（姓名、本周完成、下周计划、备注）→ 调用技能，传入知识库或文件夹 id 及字段定义 → 技能在该位置创建多维表格并建表、清占位 → Agent 回复「已在 XX 知识库创建「周报」表，可填写姓名、本周完成、下周计划、备注」。
- **最终结果**：指定知识库下新增「周报」多维表格，团队成员可在此填写周报。

**示例 3：快速建一张带示例行的表**

- **用户**：「建一个「Bug 登记」表，字段：标题、严重程度、负责人、状态；先写一行示例。」
- **Agent 实现**：解析出表名、字段及「写一行示例」→ 调用 feishu-bitable-creator 创建表与字段后，再调用「新增记录」写入一行示例数据（如标题=「示例 Bug」、严重程度=「中」、负责人=当前用户、状态=「待处理」）→ Agent 回复「已创建「Bug 登记」表并写入 1 条示例，可按此格式继续登记」。
- **最终结果**：用户得到一张可直接使用的 Bug 表，且有一行示例便于理解格式。

### 与 api-bitable 的区别

| 能力         | feishu-bitable-creator | feishu-api-bitable |
|--------------|------------------------|---------------------|
| 从零建表     | ✅ 擅长                 | 支持但非主打        |
| 清占位、批量建字段/记录 | ✅                     | 需自行组织          |
| 日常增删改查 | 非主打                 | ✅ 擅长             |

---

## 注意事项

- 创建位置（如哪个知识库、哪个文件夹）需在调用时指定或使用技能默认配置；权限不足时会报错。
- 字段类型（文本、数字、日期、人员等）需与飞书多维表格支持的类型一致，见飞书文档。
