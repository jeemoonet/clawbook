# skill-vetter：技能安全审查

[← 返回技能列表](./index.md)

---

## 一句话用途

在**安装或启用第三方技能之前**，对技能包做**安全与合规扫描**，帮助发现潜在风险（如可疑脚本、敏感信息泄露、不当权限等），适合企业规范「只允许通过审查的技能上线」。

---

## 适用场景

- **上线前检查**：新从 ClawHub / SkillHub 安装的技能，先跑一遍 vetter，再决定是否在 `skills.entries` 里设为 `enabled: true`。
- **定期审计**：对已启用技能做复查，或在技能更新后重新扫描。
- **合规**：满足「第三方代码需经安全审查」的流程要求。

---

## 安装

```bash
clawhub install skill-vetter
# 或
skillhub install skill-vetter
```

---

## 配置

- 通常无需额外 API Key；部分检查可能依赖本地规则或名单，以技能文档为准。
- 在 `~/.openclaw/openclaw.json` 的 `skills.entries["skill-vetter"]` 中设为 `enabled: true` 即可在需要时被调用；也可仅通过命令行在安装后对指定技能目录执行一次扫描（若技能支持 CLI 模式）。

---

## 怎么用

### 方式一：在安装流程中调用

安装新技能后，对技能目录执行审查（命令以技能实际提供为准）；输出会包含风险等级、发现项说明与建议。

### 方式二：让 AI 在决策前调用

在内部流程中约定：新增或更新技能时，先由负责人或 AI 调用 skill-vetter 对目标技能做扫描，根据结果决定是否启用。适合与 [安全与合规](../06-安全与合规.md) 中的「仅启用经审批技能」配合。

### 三个示例

**示例 1：新装技能上线前扫描**

- **用户**：「刚装了 summarize，你扫一遍再告诉我能不能开。」
- **Agent 实现**：识别为「对指定技能做安全审查」→ 调用 skill-vetter，传入技能路径（如 `~/.openclaw/skills/summarize`）→ 技能扫描该目录下 SKILL.md、脚本、package 依赖等，按规则检查 shell 调用、网络请求、敏感路径、依赖来源 → 输出「低风险：1 项，建议人工确认某脚本仅访问用户指定 URL」等。
- **最终结果**：用户得到一段审查结论与建议，据此决定是否在配置里将 summarize 设为 `enabled: true`。

**示例 2：发现 shell 调用后人工复核**

- **用户**：对某技能执行 skill-vetter 后，报告显示「存在执行 shell 的调用，建议人工复核」。
- **Agent 实现**：skill-vetter 在脚本中检测到 `exec`/`child_process` 等模式并标记 → 报告中给出文件名与行号（或代码片段）→ Agent 将结论与位置整理成回复，并建议「请打开该文件确认是否为预期行为（如仅执行已校验的命令）」。
- **最终结果**：用户打开对应文件复核后，确认是合法的「调用系统 summarize 命令」而非任意拼接，决定通过并启用；或发现风险后选择不启用并记录原因。

**示例 3：更新后重新审查**

- **用户**：「tavily-search 更新了，再扫一遍。」
- **Agent 实现**：对已安装的 tavily-search 目录再次调用 skill-vetter → 扫描当前文件与依赖版本 → 输出与上次对比的结果（如「仍通过」或「新增依赖 XXX，建议检查其 CVE」）→ Agent 汇总为「复测通过，可继续使用」或「建议升级某依赖后再测」。
- **最终结果**：用户确认更新后的技能仍满足安全要求，或按建议升级依赖后再次审查直至通过，满足变更后必审的流程。

### 典型检查项（概念）

- 是否包含可疑的远程请求、执行系统命令、读写敏感路径。
- 依赖与脚本是否来自可信来源、是否有已知漏洞。
- 是否要求过高权限或敏感环境变量。

具体项以 skill-vetter 当前版本为准。

---

## 注意事项

- **辅助而非替代**：vetter 是自动化辅助，不能替代人工代码审查；高风险或处理敏感数据的技能建议仍由专人看过 SKILL.md 和脚本再启用。
- **误报**：部分合法操作可能被标为需关注，需结合业务判断；发现不明项时可查阅技能官方文档或社区。
- **更新**：技能或 vetter 规则更新后，建议对已启用技能重新跑一遍审查。
