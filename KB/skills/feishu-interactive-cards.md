# feishu-interactive-cards：飞书交互卡片

[← 返回技能列表](./index.md)

---

## 一句话用途

在飞书里**发送带按钮、表单、投票、待办的交互卡片**，用户通过**点击**即可回复，无需打字，适合确认、选择、问卷和任务列表。

---

## 适用场景

- **需要用户二选一或多选一**：例如「确认删除？」【确认】【取消】、选时间/选人。
- **收集结构化输入**：表单（姓名、日期、下拉框等）、投票、问卷。
- **任务与待办**：在飞书中展示待办列表，用户勾选完成。

**原则**：只要存在「不确定」或「需要用户选择」时，优先发交互卡片而不是纯文字，体验更清晰。

---

## 安装

```bash
# ClawHub（需网络可达）
clawhub install feishu-interactive-cards

# 或使用 SkillHub（国内推荐）
skillhub install feishu-interactive-cards
```

安装后技能目录通常在工作区 `skills/feishu-interactive-cards/` 或 `~/.openclaw/skills/feishu-interactive-cards/`。

---

## 配置

### 1. 飞书凭证

与 OpenClaw 飞书通道共用即可。在 `~/.openclaw/openclaw.json` 中已配置 `channels.feishu.accounts.main`（appId、appSecret）则无需重复配置。

### 2. 回调服务（必须）

用户点击卡片后，飞书会把点击事件回调到你的服务。本技能通过**长轮询**接收回调，无需公网 IP。

在技能目录下启动回调服务（示例路径按实际安装为准）：

```bash
cd <技能目录>/scripts
node card-callback-server.js
```

保持该进程常驻；企业环境可写成 systemd/LaunchAgent 服务。

### 3. 飞书应用权限

在飞书开放平台为该应用开启**卡片回调**相关能力，并订阅 `card.action.trigger` 等事件（具体见技能自带文档或 [飞书卡片文档](https://open.feishu.cn/document/ukTMukTMukTM/uczM3QjL3MzN04yNzcDN)）。

## 怎么用

### 在对话中让 AI 发卡片

用户对机器人提出「需要确认」「多选一」「填表」等需求时，AI 应调用本技能**发送对应类型卡片**；用户在飞书中点击按钮或提交表单后，回调回到网关，再由 AI 或技能逻辑执行后续动作。

### 三个示例

**示例 1：删除前确认**

- **用户**：「帮我删掉工作区里的 test.txt。」
- **Agent 实现**：识别为破坏性操作，需用户确认 → 调用 feishu-interactive-cards，发送**确认卡**「确认删除文件 test.txt？」【确认】【取消】，并带上文件路径等状态 → 用户点击【确认】后，飞书将回调推送到本技能的 card-callback-server → 回调里校验路径并执行删除（或调用安全 API），再更新卡片为「已删除」或「删除失败：…」。
- **最终结果**：用户先在飞书里看到一张带两个按钮的卡片，点【确认】后才真正删除；若点【取消】则仅关闭卡片，文件保留。

**示例 2：团建方案投票**

- **用户**：「大家选一下本周团建方案：爬山、看电影、聚餐。」
- **Agent 实现**：识别为多选一 → 调用技能发送**投票卡**，选项为「爬山」「看电影」「聚餐」，标题为「本周团建方案」→ 用户点击某一项后，回调上报选择结果 → Agent 或技能可汇总各选项票数并更新卡片显示「当前：爬山 3 票、看电影 2 票……」，或到点后自动汇总并回复文字结果。
- **最终结果**：参与人在飞书里点选即可投票，无需打字；最终得到各选项得票或由 Agent 公布结果。

**示例 3：待办任务勾选**

- **用户**：「把「提交周报」「回复客户邮件」「更新需求文档」做成待办，我勾一个你记一个。」
- **Agent 实现**：调用技能发送**待办卡**，初始三条未勾选 → 用户勾选某条时，回调上报「某条已完成」→ 技能或 Agent 更新卡片状态（打勾），可选将完成项写入工作区或日历 → 用户可继续勾选直至全部完成。
- **最终结果**：用户在一张卡片上看到三条待办，勾选即标记完成，Agent 可据此做进度提醒或记录。

### 命令行快速测试（可选）

技能目录下通常提供 `send-card.js`，可用于测试发卡：

```bash
# 确认卡
node scripts/send-card.js confirmation "确认删除文件？" --chat-id oc_xxx

# 待办卡
node scripts/send-card.js todo --chat-id oc_xxx

# 投票卡
node scripts/send-card.js poll "团建方案" --options "爬山,看电影,聚餐" --chat-id oc_xxx
```

`chat_id` 为飞书群或私聊的会话 ID，可从 `openclaw logs --follow` 在对应会话发消息后看到。

### 卡片模板

技能一般自带示例：`confirmation-card.json`、`form-card.json`、`poll-card.json`、`todo-card.json` 等，可在此基础上改文案和字段。

---

## 注意事项

- **安全**：回调处理逻辑中不要直接把用户输入拼进系统命令，应做校验与转义；技能文档中的 Security 章节务必阅读。
- **去重**：同一按钮短时间多次点击可用 `event_id` 做去重（如 3 秒内只处理一次）。
- **失败提示**：若执行失败，应在卡片上更新为错误提示，而不是静默失败。
