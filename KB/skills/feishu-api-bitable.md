# feishu-api-bitable：飞书多维表格 API

[← 返回技能列表](./index.md)

---

## 一句话用途

对飞书**多维表格（Bitable）**做**完整的增删改查**：管理表、记录、字段，适合工单、项目看板、数据汇总与报表等需要结构化数据的场景。

---

## 适用场景

- **工单/需求管理**：在飞书多维表格里维护工单，AI 根据对话查询、新增或更新记录。
- **项目看板**：查看/更新任务状态、负责人、截止日期等字段。
- **数据汇总与报表**：按条件筛选记录、汇总数字、生成简单报表文案。

多维表格相当于「在线数据库 + 表格视图」，本技能让 AI 能通过 API 读写这些数据。

```bash
clawhub install feishu-api-bitable

```

## 配置

### 1. 飞书应用凭证

需在环境变量或 OpenClaw 配置中提供：

- `FEISHU_APP_ID`：飞书应用 App ID（形如 `cli_xxx`）
- `FEISHU_APP_SECRET`：飞书应用 App Secret

若 OpenClaw 已配置飞书通道，可在 `~/.openclaw/openclaw.json` 的 `skills.entries["feishu-api-bitable"]` 中通过 `env` 或 `apiKey` 注入，或让技能从 `channels.feishu` 读取（以技能文档为准）。

### 2. 飞书开放平台权限

应用需具备多维表格相关权限，例如：

- `bitable:app`（多维表格应用）
- 读写表格、记录、字段的权限（具体以 [飞书 Bitable API](https://open.feishu.cn/document/server-docs/docs/bitable-v1/app) 文档为准）

申请权限后需重新发布应用方可生效。

## 怎么用

### 在对话中使用

用户可在飞书中说；AI 会调用本技能对指定多维表格执行增删改查，并回复操作结果。

### 三个示例

**示例 1：新增一条需求记录**

- **用户**：「把刚才说的需求记到需求表里，标题是「用户登录优化」，负责人张三。」
- **Agent 实现**：解析出表（需求表，需事先配置 app_token + 表 id）、字段（标题、负责人）及取值 → 调用 feishu-api-bitable 的「新增记录」接口，传入各字段值（负责人可能需转为飞书 user_id）→ 技能请求飞书 Bitable API 插入一条记录 → 返回新记录 id → Agent 回复「已记入需求表，标题：用户登录优化，负责人：张三」。
- **最终结果**：多维表格「需求表」中新增一行，用户可在飞书中打开表格看到该条需求。

**示例 2：按状态筛选并排序**

- **用户**：「查一下状态为进行中的任务，按截止日期排。」
- **Agent 实现**：解析出表（任务表）、筛选条件（状态=进行中）、排序（截止日期）→ 调用技能的列表/查询接口，传入 filter 与 sort 参数 → 技能请求 Bitable API 返回符合条件的记录列表 → Agent 将结果整理成「进行中任务共 X 条：1. 任务A，截止 3/20 …」回复。
- **最终结果**：用户得到所有「进行中」任务并按截止日期排序的列表，便于排期或汇报。

**示例 3：更新工单状态**

- **用户**：「把工单 #123 的状态改成已完成。」
- **Agent 实现**：解析出记录标识（#123 或 record_id）、要更新的字段（状态=已完成）→ 调用 feishu-api-bitable 的更新记录接口，传入 record_id 及 fields 更新 → 技能请求 Bitable API 更新该行 → Agent 回复「工单 #123 已更新为已完成」。
- **最终结果**：多维表格中该工单行的「状态」变为「已完成」，用户收到确认。

### 典型能力

- **表**：创建/获取/删除表（以 API 支持为准）。
- **记录**：增加、列表（含筛选、分页）、更新、删除记录。
- **字段**：获取/创建/更新字段定义。

使用前需知道**多维表格的 app_token** 和**表 id**（在飞书多维表格 URL 或设置中可看到），或在技能中配置默认表格。

---

## 注意事项

- 多维表格的 **app_token** 与**表 id** 需要事先知道或由管理员配置；AI 无法自动枚举企业内所有表格。
- 大批量操作注意飞书 API 限频；可分批或加延迟。
- 敏感字段（如手机号、身份证）需符合企业数据合规要求。
