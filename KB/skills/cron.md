# cron：定时任务

[← 返回技能列表](./index.md)

---

## 一句话用途

让 OpenClaw 按**固定周期或指定时间**执行任务，例如每日提醒、周期报表汇总、定时同步数据等；任务在网关上**本地调度**，不依赖境外服务。

---

## 适用场景

- **每日/每周提醒**：如「每天早 9 点提醒我今日会议」「每周五下午发周报汇总」。
- **周期报表**：定时拉取数据、生成报表文案或写入飞书文档/多维表格。
- **定时同步**：按点从某系统拉数据写入飞书或内部表。

与「即时对话」不同：cron 是**到点自动跑**，无需用户当时在飞书里说话。

---

## 安装

```bash
clawhub install cron
# 或
skillhub install cron
```

部分版本可能以不同名称提供（如与 OpenClaw 内置的 cron 能力合并），以 `openclaw skills list` 和官方文档为准。

## 配置

### 1. 启用 Cron 能力

在 `~/.openclaw/openclaw.json` 中确认网关或技能层面已开启 cron 相关配置（具体字段以 [OpenClaw 文档](https://docs.openclaw.ai/automation/cron-jobs) 为准），例如：

```json5
{
  // 示例：cron 或 automation 相关配置
  // 具体键名以官方文档为准
}
```

### 2. 定义任务与周期

- **方式一**：在配置中写死 cron 表达式与要执行的动作（如调用某工具、发消息到某群）。
- **方式二**：通过 OpenClaw 的 cron 命令或管理界面添加/编辑任务，例如「每天 9:00 向 #日报群 发送今日会议提醒」。

任务执行时通常以**默认 Agent** 或指定 Agent 身份运行，可调用该 Agent 可用的一切技能（搜索、日历、多维表格等），因此可组合成「定时查日历 → 汇总 → 发飞书群」的流程。

---

## 怎么用

### 典型流程

1. **定义任务**：在 OpenClaw 的 cron 配置或界面中新增一条，填写 cron 表达式和「要做什么」。
2. **要做什么**：可以是「给某飞书群发一条消息」，或「运行某技能/工具」（如查日历、查多维表格、生成报表文案再发到群）。
3. **验证**：可先设一条几分钟后执行的任务，看网关日志或飞书是否收到预期结果。

### 三个示例

**示例 1：每天早会提醒（查日历 + 发群）**

- **用户**：在 cron 配置中新增任务「每天 8:30 执行：查我的飞书日历当天日程，并发到 #晨会 群」。
- **Agent 实现**：到点后 cron 调度器触发，以默认 Agent 身份执行「任务」→ Agent 调用飞书日历技能，查询当日日程 → 将结果整理成「今日会议：9:00 需求评审，14:00 …」→ 调用飞书发消息接口，向指定群发送该文案。
- **最终结果**：每天 8:30，#晨会 群会收到一条当日会议列表，无需人工查日历再转发。

**示例 2：每周一固定提醒**

- **用户**：配置「每周一 10:00 向 #周报 群发一条：请在本周五前填写本周周报」。
- **Agent 实现**：cron 表达式为 `0 10 * * 1`（周一 10:00）→ 到点触发后，执行动作为「向某群发固定消息」→ 网关或 Agent 调用飞书通道发送该文案到指定 chat_id。
- **最终结果**：每周一 10:00，#周报 群收到同一条提醒，保证不遗漏周报收集。

**示例 3：每周五自动汇总数据发群**

- **用户**：配置「每周五 18:00 从多维表格拉取本周工单统计，生成一段汇总文案发到 #周报 群」。
- **Agent 实现**：到点后 cron 触发 → Agent 调用飞书多维表格（或 feishu-api-bitable）技能，按「本周」筛选工单并做计数/汇总 → 将结果组织成「本周工单：新建 X、完成 Y、进行中 Z…」→ 通过飞书发到指定群。
- **最终结果**：每周五下班前，#周报 群自动收到基于多维表格的工单周报，无需人工统计和发送。

具体语法以 OpenClaw 当前版本的 cron 文档为准。

---

## 注意事项

- **网关常驻**：cron 依赖网关进程在运行；若网关重启，下一次触发会按新周期执行。
- **时区**：确认任务执行时区与预期一致（如服务器时区或配置的 TZ）。
- **权限与安全**：定时任务会以配置的 Agent 身份执行，能访问该 Agent 可见的凭证与技能；企业环境应限制可配置 cron 的人员与可见范围。
