# agent-browser：浏览器自动化

[← 返回技能列表](./index.md)

---

## 一句话用途

让 AI 通过 **Chrome DevTools Protocol（CDP）** 控制真实浏览器：打开页面、点击、填表、抓取内容等，适合内网系统、报表页、需登录后操作的场景，且**不依赖境外服务**，可在国内环境运行。

---

## 适用场景

- **内网/企业系统**：在浏览器中登录内部 OA、报表、后台，由 AI 按指令执行查询或填报。
- **信息抓取**：打开指定网页，提取正文、表格或列表，用于汇总或问答。
- **重复操作**：定时或按对话触发的填表、提交、截图等。

与「联网搜索」不同：搜索返回的是检索结果；浏览器技能是**直接操作页面**，能看到与用户一致的界面并与之交互。

## 配置

### 1. 浏览器模式

常见几种方式：

- **Managed Browser**：由 OpenClaw 启动独立 Chromium 实例，隔离且无需用户事先打开浏览器；一般需在配置中开启并指定端口（如 18800）。
- **Extension Relay**：通过 Chrome 扩展挂到已有 Chrome 标签页，可复用已登录会话；需安装扩展并配置网关地址与 Token。
- **Remote CDP**：连接远程已启动的浏览器（支持云端或内网另一台机器）。

企业内网建议优先用 **Managed Browser**，避免依赖员工本机浏览器状态。

### 2. 权限与安全

- 浏览器技能能执行页面内脚本、获取页面内容，属于**高权限**；企业环境应仅对受信任的 Agent 或群组开放。
- 建议在 [安全与合规](../06-安全与合规.md) 中配合访问控制与审计使用；敏感系统可限制在指定工作区或 bindings 内使用。

## 怎么用

### 在对话中使用

用户说出需要「打开某页、查数、填表、截图」等需求；AI 会调用浏览器技能执行导航、点击、输入、截图等操作，并回报结果或错误。

### 三个示例

**示例 1：打开内网报表并截图**

- **用户**：「打开咱们内网报表地址，截一张销售汇总的图发给我。」
- **Agent 实现**：解析出 URL（或从配置/上下文取内网报表地址）和动作「截图」→ 调用 agent-browser：先 navigate 到该地址，等待页面加载（或等待某元素出现）→ 定位到「销售汇总」区域（或全页）→ 执行 screenshot → 将图片通过飞书消息接口发给用户（或上传后发链接）。
- **最终结果**：用户在飞书里收到一张销售汇总的截图，无需自己登录内网打开页面。

**示例 2：从页面抓取当前库存数**

- **用户**：「去某某页面查一下当前库存数，发给我。」
- **Agent 实现**：解析出目标 URL 与目标信息「库存数」→ 调用浏览器技能：navigate 到该页 → 用技能提供的「提取文本」或「执行脚本」能力定位库存数字所在元素（如某 table cell 或 span）→ 读取该元素文本 → Agent 将数值整理成「当前库存：XXX」回复用户。
- **最终结果**：用户得到页面上的当前库存数字，无需自己打开页面查找。

**示例 3：在 OA 中填写日报**

- **用户**：「在 OA 里帮我填一下今天的日报。内容：完成了需求评审、写了接口文档。」
- **Agent 实现**：解析出目标系统（OA）、动作「填日报」、内容文本 → 调用浏览器：打开 OA 日报页面（若需登录则依赖已注入的 Cookie 或 Extension Relay 的已登录态）→ 在对应输入框 fill 标题、日期、内容 → 点击提交按钮 → 根据页面反馈判断成功或失败并回复用户。
- **最终结果**：OA 中提交了用户口述的日报内容，用户收到「已填写并提交」或失败原因说明。

### 使用注意

- **登录态**：Managed 模式每次是新浏览器，若目标站需登录，需在技能或流程中处理登录（如配置 Cookie、或先人工登录后由 Extension Relay 复用）。
- **稳定性**：页面结构变化可能导致原有操作失败，重要流程建议有重试或人工兜底。

---

## 注意事项

- 不要将账号密码等敏感信息写在对话或配置的明文中；能用 OAuth 或有限权限账号的尽量用。
- 遵守目标网站的使用条款与 robots 规则；仅对企业自有或已获授权的系统做自动化。
